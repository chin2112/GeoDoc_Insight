# GeoDoc_Insight

## 專案目的
自動解析文件路段並轉換為地圖上的大數據分佈洞察。

## 架構：方案 A (輕量分析型)
本專案目前採用輕量化腳本分析架構，作為前期的快速原型驗證，待確認資料複雜度與使用需求後，再由後端架構師升級為完整前後端分離服務。

## 系統核心流程
1. **Raw Date (PDF)**: 原始文件輸入
2. **Parser**: 從 PDF 精準提取「路段名稱」
3. **Geocoder**: 技術轉換「路名 → 經緯度」
4. **Analytics**: 大數據彙整與統計
5. **Dashboard**: 輸出互動式地理視覺化介面 (GIS)
